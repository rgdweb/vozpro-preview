RESUMO DAS MUDANCAS
===================

ANTES: Vercel → HostGator (processa audio) → tunnel → Gradio
DEPOIS: Vercel → tunnel → Gradio (DIRETO, audio limpo!)

O HostGator agora so serve como "quadro de avisos" da URL do tunnel.


ARQUIVOS CRIADOS
================

php-server/
  ├── get_tunnel.php       ← NOVO - Retorna URL do tunnel pro frontend
  ├── update_tunnel.php    ← ATUALIZADO - Com CORS e validacao
  └── config.php           ← LIMPO - Formato padrao

frontend/
  ├── omnivoice-direct.js  ← MODULO - Conexao direta com Gradio
  └── teste-omnivoice.html ← PAGINA DE TESTE standalone


PASSO 1 — HOSTGATOR (upload dos PHP)
=====================================

1.1. Acesse o File Manager do HostGator (cPanel)

1.2. Va para: public_html/omnivoice/

1.3. FAZER BACKUP do generate.php (renomeie para generate_backup.php)
    - Nao delete! Se precisar reverter, so renomear de volta

1.4. Upload estes 3 arquivos para public_html/omnivoice/:
    
    get_tunnel.php   (NOVO)
    update_tunnel.php (SUBSTITUIR o existente)
    config.php        (SUBSTITUIR o existente)

1.5. Verifique as permissoes dos arquivos (644 esta ok)


PASSO 2 — TESTAR O PHP
======================

2.1. Abra no navegador:
    https://sorteiomax.com.br/omnivoice/get_tunnel.php

2.2. Deve retornar algo como:
    {"status":"offline","tunnelUrl":"","message":"Nenhuma URL..."}

    Isso esta CORRETO — significa que o tunnel ainda nao esta rodando.

2.3. Na sua maquina local, execute o iniciar.bat normalmente

2.4. Quando o cloudflared subir e atualizar a URL, abra de novo:
    https://sorteiomax.com.br/omnivoice/get_tunnel.php

2.5. Deve retornar:
    {"status":"online","tunnelUrl":"https://xxx-yyy.trycloudflare.com","message":"GPU online e pronta"}


PASSO 3 — TESTAR A PAGINA STANDALONE
=====================================

3.1. Pegue o arquivo: frontend/teste-omnivoice.html

3.2. Abra DIRETAMENTE no navegador (duplo clique, ou arraste pro Chrome)
    - Nao precisa de servidor! E totalmente standalone

3.3. A pagina vai verificar automaticamente se a GPU esta online

3.4. Teste o fluxo completo:
    a) Upload um audio de referencia
    b) Digite um texto para gerar
    c) Clique em "Gerar Voz"
    d) Ouca o resultado

3.5. IMPORTANTE: Compare a qualidade com o sistema antigo!
    - Se o audio ficou limpo (sem voz "bebada"), a arquitetura nova funciona!


PASSO 4 — INTEGRAR NO VERCEL (quando o teste passar)
====================================================

4.1. Pegue o modulo: frontend/omnivoice-direct.js

4.2. No seu projeto Vercel, importe as funcoes:

    import { generateVoice, checkStatus } from './omnivoice-direct';

4.3. Substitua a chamada antiga (que ia pro generate.php) por:

    const result = await generateVoice({
        referenceAudio: meuArquivoAudio,
        text: "Texto para gerar",
        refText: "",           // transcricao do audio ref (vazio = auto)
        language: "Auto",
        speed: 1,
        autoTrim: true,        // corta para 10s no navegador
        onProgress: (msg) => console.log(msg)
    });

    // result.audioUrl - URL do audio gerado
    // result.audioBlob - Blob do audio gerado


PASSO 5 — O QUE MUDOU (resumo tecnico)
========================================

ARQUITETURA NOVA:

  Navegador ──GET──→ HostGator/get_tunnel.php ──→ URL do tunnel
  Navegador ──POST──→ tunnel/upload ──→ Gradio recebe audio limpo
  Navegador ──POST──→ tunnel/call/predict ──→ Job na GPU
  Navegador ←──SSE──── tunnel/call/predict/{id} ←── Resultado streaming

ELIMINADO:
  - generate.php processando audio no HostGator
  - FFmpeg trim/recodifica no HostGator
  - Download intermediario do audio
  - Double upload (HostGator → tunnel)
  - Degradação do audio por conversoes

GANHOS:
  - Audio chega LIMPO direto no Gradio (resolve voz "bebada")
  - ~5-8 segundos mais rapido por geracao
  - Menos carga no HostGator
  - Upload temporario (audio some apos a geracao)


ARQUIVOS QUE PODEM SER DELETADOS DEPOIS (opcional)
===================================================

Quando o sistema novo estiver funcionando no Vercel:
  - generate.php (ou renomear para generate_backup.php)
  - Qualquer pasta de audios temporarios no HostGator


PARAMETROS DO GRADIO (ja corrigidos)
=====================================

  language:  "Auto"       (detecta automaticamente)
  refText:   ""           (vazio se nao souber o transcript)
  instruct:  null         (sem instrucao especial)
  speed:     1            (velocidade normal - INT nao float)
  duration:  0            (automatico)

  Upload: multipart/form-data para /upload (NÃO base64!)
  Submit:  JSON para /call/predict com event_id streaming


TROUBLESHOOTING
===============

"GPU Offline" no teste:
  - O iniciar.bat esta rodando?
  - O cloudflared subiu? (deve mostrar URL trycloudflare.com)
  - O update_tunnel.php foi chamado? (verifique config.php)

"Upload falhou":
  - O tunnel esta respondendo? Teste: abra a URL do tunnel no navegador
  - Deve mostrar a interface do Gradio

"Erro na geracao":
  - O VozPro esta rodando na porta 7860?
  - Teste local: abra http://localhost:7860 no navegador

"Audio ainda bebado":
  - Desative o autoTrim no teste e veja se melhora
  - Tente com audio de referencia curto (5-10 segundos)
  - O audio de referencia deve ter fala clara, sem ruido

"CORS error":
  - O get_tunnel.php tem CORS habilitado
  - O Gradio tambem aceita CORS por padrao
  - Se der erro, verifique se o cloudflared esta rodando
